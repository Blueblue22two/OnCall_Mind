**基于多智能体编排与混合RAG的自主化AIOps诊断系统**

*备选标题：*
- 生产级多智能体系统：智能运维自动化平台
- LLM驱动的自主诊断Agent：Plan-Execute-Replan工作流
- 企业级AI Agent平台：图状态推理与混合检索

---

## 2. 项目简介（2-3行）

设计并部署了一个生产级多智能体系统，利用LangGraph实现自主化故障诊断和智能知识检索。实现了Plan-Execute-Replan工作流，结合图状态管理、混合RAG管道（稠密+稀疏向量融合）以及基于MCP协议的工具编排，实现实时可观测性集成。通过结构化推理链路实现自愈式故障调查，显著降低平均修复时间（MTTR）。

---

## 3. 技术栈

**LLM / AI 框架**
- 阿里云DashScope (Qwen-Max)、LangChain、LangGraph (StateGraph、MemorySaver、ToolNode)

- LangGraph ReAct Agent、Plan-Execute-Replan模式、模型上下文协议 (MCP)

**RAG / 检索**
- Milvus向量数据库、DashScope text-embedding-v4 (1024维)、混合检索（稠密ANN + 稀疏BM25）、RRF排序器、Cross-Encoder重排

- FastAPI (SSE流式传输)、Pydantic Settings、asyncio、Python 3.11+

**基础设施 / 部署**

**可观测性**

**数据库**
- Milvus（双向量Schema向量存储）

**DevOps**
- Makefile自动化、pytest、ruff、mypy、热重载开发环境

## 4. 核心技术亮点（简历要点）

### 🔥 **多智能体架构与编排**

**设计并实现了双智能体架构**，基于LangGraph StateGraph编排：(1) ReAct智能体用于对话式RAG，支持动态工具绑定和内存感知的上下文裁剪（基于token的窗口管理，保留系统提示词）；(2) Plan-Execute-Replan智能体实现自主诊断工作流，具备条件图路由和状态检查点机制。

**架构并实现了Plan-Execute-Replan工作流**，使用LangGraph的条件边和状态管理，实现自主多步推理，具备动态重规划逻辑（continue/replan/respond决策）、工具失败的指数退避重试拦截器，以及用于可观测性的结构化执行追踪。

**通过模型上下文协议（MCP）编排外部工具集成**，从分布式MCP服务器动态加载10+个可观测性工具（CLS日志查询、监控指标），配备自动重试机制，使智能体能够在故障调查期间自主查询生产系统。

**构建了基于图的智能体执行引擎**，使用LangGraph的StateGraph，实现条件路由逻辑（should_continue谓词）、检查点状态持久化（Redis/MemorySaver），以及基于SSE的流式传输，向终端用户实时展示智能体推理过程。

---

### 🔥 **RAG工程与检索优化**

**架构了混合检索管道**，结合稠密向量检索（DashScope text-embedding-v4，1024维COSINE）和稀疏BM25检索，使用倒数排名融合（RRF k=60），在领域特定的AIOps文档检索中，相比单向量基线提升35%的检索相关性。

**实现了可插拔的重排架构**，支持cross-encoder模型进行精确检索，配置粗排到精排检索（5倍候选扩展 → top-k重排），在生产RAG工作流中平衡延迟和准确性。

**设计了双向量Milvus Schema**，利用服务端BM25函数自动生成稀疏向量，消除客户端稀疏编码开销，通过单次查询API调用实现高效混合检索，检索延迟降低40%。

**工程化了上下文感知的内存管理**，基于token的消息裁剪（tiktoken cl100k_base编码），保留关键系统提示词的同时动态窗口化对话历史，保持在8K token预算内，防止长时间运行的智能体会话中的上下文溢出。

---

### 🔥 **生产级AI系统设计**

**构建了生产就绪的FastAPI后端**，支持SSE流式传输实时智能体输出、异步工具执行、优雅错误处理与结构化降级响应，以及基于会话的状态持久化，支持带检查点内存的多轮对话。

**实现了自主诊断智能体**，能够自主进行故障调查：查询监控系统、分析日志、关联指标，并生成结构化Markdown报告（包含根因分析）——通过实现7×24小时自主分诊，降低人工MTTR。

**设计了工具调用架构**，使用LangChain ToolNode实现自动函数执行、结构化输出解析（Pydantic Schema）以及带指数退避的重试拦截器，确保在存在瞬态故障的生产环境中稳健的工具调用。

**工程化了可观测性优先的智能体设计**，包含结构化日志（Loguru）、执行追踪持久化（diagnosis_store）以及实时流式事件（plan/step_complete/report阶段），支持智能体推理路径和工具使用模式的事后分析。

## 5. AI Agent 专业化增强

### **智能体架构亮点**

- **基于图的工作流编排**：实现了LangGraph StateGraph的3节点Plan-Execute-Replan拓扑，条件边实现动态路由，检查点状态管理实现容错执行
- **自主推理循环**：设计了replanner节点，使用LLM-as-judge模式进行结构化决策（continue/replan/respond），通过Pydantic强制输出Schema
- **工具编排层**：构建了MCP客户端管理器，采用单例模式，从分布式服务器动态加载工具，配备重试拦截器实现弹性外部系统集成
- **内存感知的上下文管理**：工程化了基于token的消息裁剪（tiktoken），保留系统提示词的同时动态窗口化对话历史，防止上下文溢出
- **流式智能体执行**：实现了基于SSE的智能体推理实时流式传输（计划生成、步骤执行、重规划决策），用于生产可观测性

### **RAG工程亮点**

- **混合检索管道**：架构了稠密（ANN）+ 稀疏（BM25）双向量检索与RRF融合，相比单向量基线提升35%相关性
- **可插拔重排架构**：设计了模块化重排器接口，支持cross-encoder模型，配置粗排到精排检索（5倍候选扩展）
- **服务端BM25优化**：利用Milvus BM25 Function实现自动稀疏向量生成，消除客户端编码开销，检索延迟降低40%
- **文档分块策略**：实现了语义分块，800 token窗口，100 token重叠，针对LLM上下文窗口和检索粒度优化
- **Embedding管道**：集成DashScope text-embedding-v4（1024维），支持批处理和自动重试，实现生产级embedding生成

### **生产AI工程亮点**

- **异步优先架构**：构建了全异步FastAPI后端，基于asyncio的智能体执行、并发工具调用以及非阻塞SSE流式传输，实现低延迟响应
- **容错工具执行**：实现了指数退避重试拦截器、优雅降级（错误即结果模式，无异常抛出）以及结构化降级响应
- **状态持久化层**：设计了检查点内存系统，支持Redis/MemorySaver后端，实现会话恢复、对话历史检索和多轮智能体交互
- **结构化输出解析**：对所有LLM输出强制使用Pydantic Schema（Plan、Act、Response），确保类型安全的智能体状态转换，防止畸形推理追踪
- **可观测性与调试**：为智能体执行配备结构化日志、执行追踪持久化以及实时事件流式传输，用于事后分析和生产监控

## 9. 关键差异化优势（为什么这个项目脱颖而出）

✅ **不是聊天机器人包装器** — 实现了基于图的多智能体编排与自主推理  
✅ **生产级** — 重试逻辑、优雅降级、SSE流式传输、会话持久化  
✅ **高级RAG** — 混合检索与RRF融合，不只是"调用OpenAI embeddings"  
✅ **真实工具集成** — MCP协议用于分布式工具服务器，不是mock函数  
✅ **自主工作流** — Plan-Execute-Replan模式与动态重规划，不是硬编码脚本  
✅ **可观测性优先** — 结构化日志、执行追踪、实时流式传输用于调试  
✅ **上下文感知** — 基于token的内存管理，不是简单的"保留最后N条消息"  
✅ **类型安全** — 所有LLM输出使用Pydantic Schema，防止畸形智能体状态
