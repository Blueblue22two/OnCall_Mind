# TLS 证书过期告警处理方案

## 告警名称
- **告警名**: `CertificateExpiry`
- **告警级别**: 紧急
- **触发条件**: TLS 证书距离过期不足7天

## 问题描述
当TLS/SSL证书即将过期或已过期时，可能导致：
- 服务间TLS握手失败
- 用户浏览器显示安全警告
- 数据传输不安全
- 业务中断

## 排查步骤

### 步骤1: 获取当前时间
**工具**: `get_current_time`
**目的**: 确定告警发生的时间范围，用于后续日志查询

### 步骤2: 查询系统日志
**工具**: `search_log` (CLS)
**参数要求**:
- **地域**: `ap-guangzhou` (广州)
- **日志主题**: `system-metrics`
- **时间范围**: 最近30分钟
- **查询条件**: `event:certificate_expiry OR level:ERROR`

**查询示例**:
```
地域: ap-guangzhou
日志主题: system-metrics
时间范围: [当前时间-30分钟] 到 [当前时间]
查询语句: event:certificate_expiry OR level:ERROR
```

### 步骤3: 分析证书信息
从日志中提取以下信息：
- 证书的域名
- 证书的有效期
- 证书的颁发者
- 证书的状态（即将过期/已过期）

### 步骤4: 查询应用日志
**工具**: `search_log` (CLS)
**参数要求**:
- **地域**: `ap-guangzhou`
- **日志主题**: `application-logs`
- **时间范围**: 与告警时间一致
- **查询条件**: `certificate_expiry OR ssl_handshake_failure`

**查询示例**:
```
地域: ap-guangzhou
日志主题: application-logs
时间范围: [当前时间-30分钟] 到 [当前时间]
查询语句: certificate_expiry OR ssl_handshake_failure
```

### 步骤5: 查询监控指标确认服务影响
**工具**: `query_cpu_metrics` (Monitor)
**参数要求**:
- **服务名**: 受证书影响的业务服务名
- **时间范围**: 最近30分钟

通过日志中的证书信息和监控指标，交叉确认证书过期对业务请求的影响范围。

## 常见原因分析

### 原因1: 证书到期未续
**特征**:
- 证书有效期即将结束
- 日志中有证书到期的警告
- 应用出现SSL握手失败

**处理方案**:
1. **立即续签**:
   - 使用证书管理工具（如Let's Encrypt）续签证书
   - 更新证书文件到服务器
   - 重启相关服务使新证书生效

2. **验证更新**:
   - 确认新证书已正确安装
   - 测试SSL握手是否成功

3. **设置提醒**:
   - 设置证书到期前30天的提醒
   - 定期检查证书有效期

### 原因2: 证书配置错误
**特征**:
- 证书文件路径配置错误
- 证书链不完整
- 证书格式不正确

**处理方案**:
1. **检查配置**:
   - 检查应用配置文件中的证书路径
   - 确认证书链完整
   - 验证证书格式（PEM, DER等）

2. **更新配置**:
   - 修正配置文件中的错误
   - 重新加载配置
   - 重启相关服务

3. **测试配置**:
   - 通过检查应用日志中的 SSL 握手记录确认证书配置正确
   - 确认证书绑定域名与实际访问域名一致

### 原因3: 证书被吊销
**特征**:
- 证书状态为吊销
- 应用出现SSL握手失败
- 日志中有证书吊销的警告

**处理方案**:
1. **获取新证书**:
   - 联系证书颁发机构（CA）获取新的证书
   - 更新证书文件到服务器
   - 重启相关服务

2. **验证更新**:
   - 确认新证书已正确安装
   - 测试SSL握手是否成功

3. **监控证书状态**:
   - 定期检查证书状态
   - 设置证书吊销的告警

### 原因4: 证书密钥泄露
**特征**:
- 证书密钥泄露
- 应用出现SSL握手失败
- 日志中有证书密钥泄露的警告

**处理方案**:
1. **撤销证书**:
   - 联系证书颁发机构（CA）撤销现有证书
   - 生成新的密钥对
   - 获取新的证书

2. **更新证书**:
   - 更新证书文件到服务器
   - 重启相关服务

3. **加强安全措施**:
   - 加强密钥存储的安全性
   - 定期更换密钥
   - 监控密钥访问日志

### 原因5: 证书不匹配
**特征**:
- 证书域名与实际域名不匹配
- 应用出现SSL握手失败
- 日志中有证书不匹配的警告

**处理方案**:
1. **检查域名**:
   - 确认证书中的域名与实际域名一致
   - 如果不一致，申请新的证书

2. **更新证书**:
   - 更新证书文件到服务器
   - 重启相关服务

3. **验证配置**:
   - 测试SSL握手是否成功
   - 确认证书配置正确

## 紧急处理措施

### 立即操作（5分钟内）
1. **续签证书**:
   - 使用自动化工具（如Let's Encrypt）快速续签证书
   - 更新证书文件到服务器
   - 重启相关服务

2. **临时切换**:
   - 如果无法立即续签，暂时使用自签名证书
   - 通知用户可能的安全风险

### 短期措施（30分钟内）
1. **验证更新**:
   - 确认新证书已正确安装
   - 测试SSL握手是否成功

2. **检查配置**:
   - 检查证书路径和配置文件
   - 确认证书链完整

3. **监控状态**:
   - 持续监控证书状态
   - 设置证书到期前的提醒

### 长期优化
1. **自动化续签**:
   - 使用自动化工具定期续签证书
   - 设置证书到期前的自动提醒

2. **加强安全**:
   - 加强密钥存储的安全性
   - 定期更换密钥
   - 监控密钥访问日志

3. **完善文档**:
   - 更新运维文档
   - 定期培训相关人员

## 验证步骤
1. 确认证书已正确安装
2. 测试SSL握手是否成功
3. 检查应用日志无SSL握手失败记录
4. 持续监控30分钟确保稳定
